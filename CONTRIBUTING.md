# Contributing

All changes follow the GeoAware engineering lifecycle.

1. Define user value and affected system boundaries.
2. Confirm GeoAware Core and product-passport alignment.
3. Make a focused branch and reviewable diff.
4. Run `pytest -q`, `python scripts/validate_static.py`, and `node --check app.js`.
5. Include risks, validation evidence, deployment impact, and ADR impact in the pull request.
6. Resolve security, privacy, data-integrity, accessibility, and deployment blockers before merge.
7. Obtain Project Architect approval for product-identity, architecture, or visible-feature changes.

Formatting-only preferences are non-blocking unless they affect comprehension or maintainability.
